## Design Context

### Users

**Primary**: Norwegian chiropractors (kiropraktorer) running solo or small-group practices.
**Secondary**: Clinic receptionists managing bookings and patient flow.
**Tertiary**: Patients using the portal for booking, messaging, and exercise programs.

**Context**: Desktop-first (Electron app). Used in clinical settings between patient appointments — often with 5-10 minutes between interactions. Must be fast to navigate, clear at a glance, and never ambiguous about clinical data. Patients access a portal on their phones.

**Job to be done**: Run an entire chiropractic practice — scheduling, clinical notes (SOAP), billing, patient communication, exercise prescriptions, AI-assisted diagnostics — from a single application without switching tools.

### Brand Personality

**Three words**: Clinical, Warm, Efficient

- **Clinical**: Teal primary signals trust and medical authority. Clean layouts, precise data presentation, WCAG AA compliance by law.
- **Warm**: Nordic-inspired warmth (amber accents, warm off-white backgrounds, rounded corners). Approachable without being playful.
- **Efficient**: Minimal clicks, fast navigation, information-dense without feeling cluttered. Dashboard-driven workflow.

**Emotional goals**: Confidence (clinicians trust the data), Calm (warm surfaces reduce visual fatigue during long sessions), Competence (the UI feels professional and purpose-built).

### Aesthetic Direction

**Visual tone**: Scandi-Clinical — Scandinavian minimalism meets healthcare precision. Warm neutrals, generous whitespace, soft shadows, teal accents for interactive elements.

**References**:

- Scandinavian design principles: clean lines, natural warmth, function over decoration
- shadcn/ui component architecture: HSL token system, Radix UI primitives, Tailwind-first
- Medical EHR conventions: data-dense tables, status badges with icons (not color alone), audit-friendly interactions

**Anti-references**:

- NOT flashy SaaS dashboards (no gradients, no neon colors)
- NOT cold clinical (no blue/gray sterile hospital feel)
- NOT playful consumer apps (no rounded bubble shapes, no emoji-heavy UI)

**Theme**: Light mode primary, dark mode fully supported. Class-based toggle (`darkMode: ['class']`).

### Design Principles

1. **Token-first**: Every color, spacing, radius, and shadow comes from `tokens.css`. No hardcoded values in components (except chart visualization arrays).
2. **Accessibility is law**: WCAG 2.1 AA is legally required in Norway. 4.5:1 contrast for text, 3:1 for UI components. Color-independent status indicators. Norwegian-language labels on all interactive elements.
3. **Warm precision**: Surfaces lean warm (amber undertones in backgrounds, borders). The teal primary anchors clinical trust. Dark mode preserves warmth.
4. **Progressive density**: Dashboard shows KPIs at a glance. Drill down reveals full detail. Never force users to scroll to find critical information.
5. **Norwegian first**: All patient-facing text in Bokmal (nb-NO). Informal (du/deg). Medical terms in Norwegian with Latin/English in parentheses.

### Typography (current — pre-Sprint 7A)

- **Font**: System stack (no custom typeface)
- **Scale**: Fixed rem (`--text-xs` through `--text-display`)
- **Weights used**: regular, medium (500), semibold (600), bold (700)
- **Planned**: Inter web font + fluid `clamp()` scale

### Spacing

| `--space-lg`  | 24px  | Between sections             |
| `--space-xl`  | 32px  | Page margins                 |
| `--space-2xl` | 48px  | Major section breaks         |

### Border Radius

Base `--radius: 0.5rem` (8px). Tailwind extends: `sm` (4px), `md` (6px), `lg` (8px), `xl` (12px), `2xl` (16px).

### Shadows

Soft shadow system: `shadow-soft-sm`, `shadow-soft`, `shadow-soft-md`, `shadow-soft-lg`. Low opacity for Scandinavian lightness.

### Component Library

- **Base**: 22+ components in `frontend/src/components/ui/` (Button, Card, Input, StatusBadge, StatCard, EmptyState, Skeleton, ConfirmDialog, LoadingButton, etc.)
- **Primitives**: Radix UI (Accordion, Dialog, Dropdown, Select, Tabs, Toast)
- **Icons**: Lucide React
- **Charts**: Recharts (with hardcoded hex colors — migration target)
- **i18n**: Custom `useTranslation` hook (NOT react-i18next)

### Dark Mode

Class-based toggle (`.dark` on `<html>`). Every semantic token has light + dark variant in `tokens.css`. Components use `dark:` Tailwind prefix for overrides.

### Layout Conventions

- Page container: `p-4 sm:p-6 max-w-7xl mx-auto`
- Stats grid: `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-6 gap-4`
- Section spacing: `space-y-6` between major sections
- Card padding: `p-5` or `p-6`
